-CurLine — это сайт на Django, который автоматически собирает и сохраняет статьи с других ресурсов, распределяет их по категориям и подкатегориям и отображает их на главной странице в виде бесконечной ленты; зарегистрированные пользователи могут читать материалы, ставить лайки, добавлять статьи в избранное, видеть дату публикации, дату обновления (если статья была доработана) и статистику — количество просмотров и лайков; администраторы получают доступ к панели, где они могут вручную добавлять, редактировать и удалять статьи.Фронтенд генерила Webcrumbs + Cursor AI.
+# CurLine
 
-Технические детали:
-Проект создан на Python 3 и Django 5. Для парсинга используются библиотеки BeautifulSoup и Requests. Модели включают Article, Category, User и Like с соответствующими связями. Автоматический парсинг статей реализован через python файл, запускаемые по расписанию. Бесконечная лента работает через AJAX-запросы к Django REST-API. Аутентификация пользователей базируется на стандартной Django-системе с разграничением ролей (user/admin).
+CurLine — это Django‑приложение, которое собирает статьи с внешних источников, хранит их в базе и показывает пользователям в виде бесконечной ленты. Зарегистрированные пользователи могут читать материалы, ставить лайки и добавлять статьи в избранное, а администраторы управляют контентом через панель Django Admin.
+
+## Возможности
+
+- Автоматический парсинг статей с `towardsdatascience.com` через management-команду `main`.
+- Лента статей с пагинацией и AJAX‑подгрузкой (реализовано на уровне view/шаблонов).
+- Просмотр статьи по slug (или fallback по ID), поиск по заголовкам.
+- Лайки и избранное для авторизованных пользователей.
+- Регистрация, вход/выход, сброс пароля через стандартные Django‑представления.
+- Административная панель для управления статьями и категориями (Django Admin).
+
+## Технологии
+
+- **Backend:** Python 3, Django 5.1
+- **База данных:** SQLite (по умолчанию)
+- **Парсинг:** requests + BeautifulSoup
+- **Дополнительно:** кастомная модель пользователя и связи для лайков/избранного
+
+## Быстрый старт
+
+### 1) Установка зависимостей
+
+```bash
+python -m venv .venv
+source .venv/bin/activate  # Windows: .venv\\Scripts\\activate
+pip install -r requirements.txt
+```
+
+### 2) Переменные окружения
+
+Создайте файл `.env` в корне проекта и укажите минимум:
+
+```env
+SECRET_KEY=your-secret-key
+EMAIL_USER=your-email@gmail.com
+EMAIL_PASS=your-email-password
+```
+
+Переменные используются в настройках проекта для секретного ключа и SMTP‑почты.
+
+### 3) Миграции и суперпользователь
+
+```bash
+python manage.py migrate
+python manage.py createsuperuser
+```
+
+### 4) Запуск приложения
+
+```bash
+python manage.py runserver
+```
+
+Проект будет доступен по адресу `http://127.0.0.1:8000/`.
+
+## Парсинг статей
+
+Основная команда:
+
+```bash
+python manage.py main --pages 3
+```
+
+- `--pages` задает количество страниц, с которых нужно собрать ссылки на статьи.
+
+Дополнительно есть скрипт-обертка `run_parser.py`, который запускает команду парсинга из Python‑скрипта (можно использовать для Cron/Task Scheduler).
+
+Планировщик:
+
+```bash
+python manage.py schedule_parser
+```
+
+- Запускает парсинг при старте и повторяет задачу по расписанию (понедельник 09:00).
+
